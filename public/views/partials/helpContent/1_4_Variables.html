1.4  Variables

So far, our service has been pretty boring and static. It's the same for every user, it doesn't contain any variable content.

Let's extend our #profile service to save the user's name and repeat it back later.

   
{
    "content": [{
        "ref": "/profile/index.json",
        "type": "input",
        "menuRef": "/profile/menu.json",
        "header": "** #profile **",
        "content": {
            "type": "string",
            "description": "What is your first name?",
            "var": "firstname" 
        }
   }, {
        "menuRef": "profile/menu.json",
        "type": "menu",
        "header": "** #profile menu **\nHi {{firstname}}!\nWhat is your favourite colour?", 
        "content": [{
            "ref": "profile/red.json",
            "type": "linkStatic",
            "shortcut": "red",
            "description": "I like red"
        }, {
            "ref": "profile/blue.json",
            "type": "linkStatic",
            "shortcut": "blue",
            "description": "I prefer blue"
        }, {
            "type": "skip",
            "shortcut": "skip",
            "description": "skip"
        }]
    },
   {
    	"type": "end",
    	"ref": "#onem"
    }]
}

Now our profile service is starting to take shape. It will save whatever the user enters as their first name in the variable firstname. Then in the next step, when the menu is displayed, we tell the simulator to recall the variable and display this using the double-curly bracket notation {{expression}}
